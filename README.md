# 602 Ops Portal

A full-stack operations management MVP for a coffee shop and coffee roastery company with three coffee shops and one roastery.

## Stack

- Next.js App Router
- TypeScript
- PostgreSQL
- Prisma ORM
- Tailwind CSS
- NextAuth credentials authentication
- Role-based permissions
- Render-ready deployment

## MVP Modules

- Login and protected routes
- Role-based access for Owner, Area Manager, Store Manager, Shift Lead, and Staff
- Employee profile management
- Monthly staff schedule calendar
- Training materials library with categories
- Training quiz/test data model
- HR document checklist per employee
- Inventory tracking with par levels
- Order request system
- Manager dashboard
- Admin overview

## Local Setup

1. Install dependencies:

```bash
npm install
```

2. Copy the example environment file:

```bash
cp .env.example .env
```

3. Set `DATABASE_URL`, `AUTH_SECRET`, and `AUTH_URL` in `.env`.

4. Run migrations:

```bash
npm run prisma:migrate
```

5. Seed the database:

```bash
npm run prisma:seed
```

6. Start the dev server:

```bash
npm run dev
```

Open `http://localhost:3000`.

## Render Deployment

This project includes a `render.yaml` Blueprint. Render can create both the web service and PostgreSQL database from the GitHub repository.

1. Push this folder to a GitHub repository.
2. In Render, choose **New > Blueprint**.
3. Connect the GitHub repository.
4. Render will read `render.yaml` and create:
   - `602-ops-portal` web service
   - `602-ops-portal-db` PostgreSQL database
5. Set `AUTH_URL` to your Render service URL:

```txt
AUTH_URL=https://<your-render-service>.onrender.com
```

`DATABASE_URL` is wired from the Render database automatically, and `AUTH_SECRET` is generated by Render.

The Blueprint uses this build command:

```bash
npm install && npm run build
```

The Blueprint uses this pre-deploy command:

```bash
npm run prisma:deploy && npm run prisma:seed
```

The Blueprint uses this start command:

```bash
npm run start
```

The seed script is idempotent and runs during pre-deploy so the initial demo accounts exist on Render.

## GitHub Integration

This project includes `.github/workflows/ci.yml`.

The workflow runs on pushes and pull requests to `main`:

- Installs dependencies
- Starts a PostgreSQL service
- Generates Prisma client
- Applies migrations
- Runs lint
- Builds the Next.js app

Render will automatically deploy new commits after GitHub checks pass.

## Notes

- Secrets live in environment variables only.
- The Prisma schema includes NextAuth adapter tables plus operations tables.
- The first version uses credentials auth with seeded passwords. For production, add password reset, invitation flow, audit logging, and optional SSO.
